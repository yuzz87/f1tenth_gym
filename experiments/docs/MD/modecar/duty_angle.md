# ステアリングduty比・角度実測データ

## 目的

ステアリングサーボへ出力するPWM duty比[%]と、実車の前輪ステア角[deg]、[rad]の対応関係を記録する。

このデータは、目標ステア角[rad]をステアリングduty比[%]へ変換するための校正値として使う。

## 前提

- PWM周波数は `70 Hz` とする。
- duty比はステアリングサーボへ出力するPWM duty比[%]である。
- 測定時の角度は度[deg]で記録されている。
- 制御プログラム内部では角度をラジアン[rad]で扱う。
- 元データの最初の数値列は、実測時に出力したステアリングduty比[%]として扱う。
- `右[deg]` と `左[deg]` は左右の測定値として扱い、変換式には平均角度を使う。

## 実測データ

元データは `scripts/machine.py` のメモから転記した。

| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |

## 度[deg]の近似式

`x` をステアリングduty比[%]、`y` をステア角[deg]とする。

```text
y = -10.702 * x + 115.65
```

コードで使う名前にすると次である。

```text
steer_angle_deg = -10.702 * steering_duty_percent + 115.65
```

## 度[deg]の逆変換

ステア角[deg]からステアリングduty比[%]を求める式は次である。

```text
x = (115.65 - y) / 10.702
```

コードで使う名前にすると次である。

```text
steering_duty_percent = (115.65 - target_steer_deg) / 10.702
```

## ラジアン[rad]の近似式

制御プログラムではラジアン[rad]を使うため、度[deg]の式をラジアン[rad]へ変換して使う。

```text
steer_angle_rad = -0.186785136654 * steering_duty_percent + 2.018473280947
```

## ラジアン[rad]の逆変換

目標ステア角[rad]からステアリングduty比[%]を求める式は次である。

```text
steering_duty_percent = (target_steer_rad - 2.018473280947) / -0.186785136654
```

現在の `steering_check.py` ではこの逆変換を使って、確認用のduty比[%]を計算している。

## 中立値の注意

実測データでは、中心の実測duty比は次である。

```text
measured_neutral_duty_percent = 10.895 %
```

一方、線形式から `0 deg` または `0 rad` を逆変換すると次になる。

```text
fit_zero_angle_duty_percent = 10.806391330 %
```

差は次である。

```text
10.895 - 10.806391330 = 0.088608670 %
```

PWM周波数 `70 Hz` では、この差は約 `12.66 us` である。

そのため、制御で使う「目標角度 `0 rad` のduty比」と、緊急停止や手動中立で使う「実測中立duty比」は区別する。

現在の中立候補:

```text
steering neutral duty = 10.895 %
```

現在の線形式による `0 rad` 指令:

```text
target_steer_rad = 0.0 rad -> steering_duty_percent = 10.806391330 %
```

## 使用上の注意

- 現在の確認範囲は概ね `-18 deg` から `18 deg`、つまり約 `-0.314 rad` から `0.314 rad` である。
- duty比が大きくなるほど、ステア角は負方向へ変化する。
- このデータは静的な前輪角度の測定であり、実走時のタイヤすべり、サーボ応答遅れ、リンケージのバックラッシュは含まない。
- 実車へ出力するときはESCを中立にし、駆動輪を浮かせて確認する。
